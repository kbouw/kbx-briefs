# Daily Intelligence Brief — April 25, 2026

## AI/ML

### 1. 🔴 llm 0.31

llm 0.31 adds support for OpenAI's new GPT-5.5 model plus fine-grained control options including verbosity levels (low/medium/high) for GPT-5+ models and image detail settings. The CLI tool now lets you tune model behavior for specific use cases rather than relying on default configurations.

**Source:** [Simon Willison's Weblog](https://simonwillison.net/2026/Apr/24/llm/#atom-everything)
**Tags:** llm, GPT-5.5, OpenAI, CLI tools

### 2. 🔴 GPT-5.5 prompting guide

OpenAI recommends treating GPT-5.5 as entirely new model family requiring fresh prompts from scratch rather than migrating existing ones, plus suggests sending short user-visible updates before multi-step tasks to prevent users thinking the model crashed during long processing times.

**Source:** [Simon Willison's Weblog](https://simonwillison.net/2026/Apr/25/gpt-5-5-prompting-guide/#atom-everything)
**Tags:** GPT-5.5, prompt engineering, OpenAI API

### 3. 🔴 Quoting Romain Huet

OpenAI unified Codex with the main GPT model starting in GPT-5.4, eliminating the separate coding-focused model line, with GPT-5.5 delivering significant improvements in agentic coding and computer automation tasks. This confirms there won't be a standalone GPT-5.5-Codex release, signaling OpenAI's shift toward general-purpose models that excel at coding rather than specialized variants.

**Source:** [Simon Willison's Weblog](https://simonwillison.net/2026/Apr/25/romain-huet/#atom-everything)
**Tags:** GPT-5.4, GPT-5.5, Codex, agentic coding
