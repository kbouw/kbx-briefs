## AI/ML

### 1. AI changed what matters when you build software

AI has shifted the software development bottleneck from code production to architectural decision-making—knowing which code should exist, why, and how to structure it becomes the scarce skill as AI can rapidly generate any requested functionality. The real value now lies in using AI as a thinking partner to pressure-test your reasoning rather than just a code generator.

**Source:** [Coen Jacobs](https://coenjacobs.com/blog/ai-changed-what-matters/)
**Tags:** AI agents, software architecture, development workflow, decision-making

### 2. Thoughts on slowing the fuck down

Mario Zechner, creator of the Pi agent framework, warns that coding agents generate "booboos" (errors) so fast they compound into unmaintainable codebases before developers realize what happened. He recommends setting daily limits on agent-generated code that match your actual review capacity and writing architecture/APIs by hand.

**Source:** [Simon Willison's Weblog](https://simonwillison.net/2026/Mar/25/thoughts-on-slowing-the-fuck-down/#atom-everything)
