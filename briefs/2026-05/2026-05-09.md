### 1. AI is breaking two vulnerability cultures

Jeff Kaufman argues AI is disrupting both responsible disclosure (researchers may withhold AI vulnerabilities indefinitely) and coordinated disclosure (AI companies may never acknowledge or fix reported issues) cultures, creating a accountability vacuum where neither traditional security frameworks apply effectively.

### 2. AI makes weak engineers less harmful

AI coding tools like Claude Code have dramatically raised the floor for weak engineers—instead of producing net-negative code that breaks things, they now output functional but flawed code that's manageable to review. This creates a new dynamic where the weakest engineers effectively become "Claude-over-Slack" interfaces, which is better than sabotage but signals these roles may be eliminated as companies question what value engineers add to AI.

**Source:** [Sean Goedecke](https://seangoedecke.com/ai-makes-weak-engineers-less-harmful/)
**Tags:** Claude Code, software engineering, code quality, developer productivity
